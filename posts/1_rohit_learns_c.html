the thing that makes c different from other languages like java or c++ is the manual memory management aspect of it. i'll be honest, i only finished one lesson which covered pointers and basic syntax, but hey that is the topic they all talk about when it comes to c and memory management. so let's get on with it.

to understand pointers, we need to first understand how memory works, ie., how data from your code is stored in the memory when you run your program. without going too deep, let's look at the two types of memory storage that are relevant here:

  1. stack
  2. heap

stack-based memory allocation:
stack is that region in the memory where local variables and functions are stores. each thread has an independent reserved stack. every time a function is called, it gets placed on the top of the stack until it returns. if a function is called inside another function that is already being executed (such as nested functions or anonymous functions), it gets placed on top of the function that is already running. the function that is on the top of the stack must always return before anything that is under it on the stack. this principle is known as Last-in-First-out (LIFO). LIFO is also a principle used in cost accounting, but fuck if i remember what it actually did. but here it means the last thing to enter the stack must be the first to go out. any variables declared inside the current executing function will only remain in memory while the function is on the stack. once the function has returned, any variables within the local scope will be lost. that means the data is not longer accessible.
